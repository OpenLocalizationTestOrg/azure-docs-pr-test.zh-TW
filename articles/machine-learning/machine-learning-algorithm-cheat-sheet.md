## <a name="notes-and-terminology-definitions-for-the-machine-learning-algorithm-cheat-sheet"></a><span data-ttu-id="dee17-119">機器學習演算法的注意事項與詞彙定義</span><span class="sxs-lookup"><span data-stu-id="dee17-119">Notes and terminology definitions for the machine learning algorithm cheat sheet</span></span>

* <span data-ttu-id="dee17-120">這份演算法小祕技中提供的建議是近似經驗法則。</span><span class="sxs-lookup"><span data-stu-id="dee17-120">The suggestions offered in this algorithm cheat sheet are approximate rules-of-thumb.</span></span> <span data-ttu-id="dee17-121">您可以屈從有些建議，也可以公然違反有些建議。</span><span class="sxs-lookup"><span data-stu-id="dee17-121">Some can be bent, and some can be flagrantly violated.</span></span> <span data-ttu-id="dee17-122">其主要目的是建議起點。</span><span class="sxs-lookup"><span data-stu-id="dee17-122">This is intended to suggest a starting point.</span></span> <span data-ttu-id="dee17-123">別擔心您的資料上會出現數種演算法間勢均力敵的競爭。</span><span class="sxs-lookup"><span data-stu-id="dee17-123">Don’t be afraid to run a head-to-head competition between several algorithms on your data.</span></span> <span data-ttu-id="dee17-124">唯有了解每個演算法的原則，以及了解產生您的資料的系統。</span><span class="sxs-lookup"><span data-stu-id="dee17-124">There is simply no substitute for understanding the principles of each algorithm and understanding the system that generated your data.</span></span>

* <span data-ttu-id="dee17-125">每個機器學習服務演算法都有自己的風格或*歸納偏差*。</span><span class="sxs-lookup"><span data-stu-id="dee17-125">Every machine learning algorithm has its own style or *inductive bias*.</span></span> <span data-ttu-id="dee17-126">對於特定問題，適合的演算法可能有數個，但其中一個演算法可能會比其他演算法更適合。</span><span class="sxs-lookup"><span data-stu-id="dee17-126">For a specific problem, several algorithms may be appropriate and one algorithm may be a better fit than others.</span></span> <span data-ttu-id="dee17-127">但不一定能夠事先知道哪一種最適合。</span><span class="sxs-lookup"><span data-stu-id="dee17-127">But it's not always possible to know beforehand which is the best fit.</span></span> <span data-ttu-id="dee17-128">在這類情況下，小祕技中會一起列出數個演算法。</span><span class="sxs-lookup"><span data-stu-id="dee17-128">In cases like these, several algorithms are listed together in the cheat sheet.</span></span> <span data-ttu-id="dee17-129">適當策略就是嘗試一個演算法，而如果結果還不滿意，就嘗試其他演算法。</span><span class="sxs-lookup"><span data-stu-id="dee17-129">An appropriate strategy would be to try one algorithm, and if the results are not yet satisfactory, try the others.</span></span> <span data-ttu-id="dee17-130">以下是 [Cortana Intelligence 資源庫](http://gallery.cortanaintelligence.com/)中的實驗範例，該實驗對相同的資料嘗試數種演算法，並比較其結果：[比較多類別分類器：字母辨識](http://gallery.cortanaintelligence.com/Details/a635502fc98b402a890efe21cec65b92)。</span><span class="sxs-lookup"><span data-stu-id="dee17-130">Here’s an example from the [Cortana Intelligence Gallery](http://gallery.cortanaintelligence.com/) of an experiment that tries several algorithms against the same data and compares the results: [Compare Multi-class Classifiers: Letter recognition](http://gallery.cortanaintelligence.com/Details/a635502fc98b402a890efe21cec65b92).</span></span>

* <span data-ttu-id="dee17-131">機器學習有三個主要類別：**經過指導的學習**、**未經指導的學習**和**增強式學習**。</span><span class="sxs-lookup"><span data-stu-id="dee17-131">There are three main categories of machine learning: **supervised learning**, **unsupervised learning**, and **reinforcement learning**.</span></span>

  * <span data-ttu-id="dee17-132">在**經過指導的學習**中，每個資料點都會加上標籤或與感興趣的類別或值產生關聯。</span><span class="sxs-lookup"><span data-stu-id="dee17-132">In **supervised learning**, each data point is labeled or associated with a category or value of interest.</span></span>  <span data-ttu-id="dee17-133">分類標籤的範例就是將影像指派為「貓」或「狗」。</span><span class="sxs-lookup"><span data-stu-id="dee17-133">An example of a categorical label is assigning an image as either a ‘cat’ or a ‘dog’.</span></span>  <span data-ttu-id="dee17-134">值標籤的範例則是與中古汽車相關聯的銷售價格。</span><span class="sxs-lookup"><span data-stu-id="dee17-134">An example of a value label is the sale price associated with a used car.</span></span> <span data-ttu-id="dee17-135">監督式學習的目標是研究許多像是這些已加上標籤的範例，然後能夠做出有關未來資料點的預測。</span><span class="sxs-lookup"><span data-stu-id="dee17-135">The goal of supervised learning is to study many labeled examples like these, and then to be able to make predictions about future data points.</span></span> <span data-ttu-id="dee17-136">例如，識別包含正確動物的新相片，或者針對其他中古汽車指派精確的銷售價格。</span><span class="sxs-lookup"><span data-stu-id="dee17-136">For example, identifying new photos with the correct animal or assigning accurate sale prices to other used cars.</span></span> <span data-ttu-id="dee17-137">這是常見且實用的機器學習服務類型。</span><span class="sxs-lookup"><span data-stu-id="dee17-137">This is a popular and useful type of machine learning.</span></span> <span data-ttu-id="dee17-138">除了 [K-Means 叢集][k-means-clustering]以外，Azure Machine Learning 中的所有模組都是經過指導的學習演算法。</span><span class="sxs-lookup"><span data-stu-id="dee17-138">All of the modules in Azure Machine Learning are supervised learning algorithms except for [K-Means Clustering][k-means-clustering].</span></span>

  * <span data-ttu-id="dee17-139">在**未監督的學習**中，資料點沒有與其相關聯的標籤。</span><span class="sxs-lookup"><span data-stu-id="dee17-139">In **unsupervised learning**, data points have no labels associated with them.</span></span> <span data-ttu-id="dee17-140">然而，未經指導的學習演算法的目標在於以某種方式組織資料或描述其結構。</span><span class="sxs-lookup"><span data-stu-id="dee17-140">Instead, the goal of an unsupervised learning algorithm is to organize the data in some way or to describe its structure.</span></span> <span data-ttu-id="dee17-141">這可能表示將資料劃分為叢集 (如 K-Means 所為)，或尋找各種查看複雜資料的方式，使其變得更簡單。</span><span class="sxs-lookup"><span data-stu-id="dee17-141">This can mean grouping it into clusters, as K-means does, or finding different ways of looking at complex data so that it appears simpler.</span></span>

  * <span data-ttu-id="dee17-142">在**增強式學習**中，演算法需要選擇一個動作來回應每個資料點。</span><span class="sxs-lookup"><span data-stu-id="dee17-142">In **reinforcement learning**, the algorithm gets to choose an action in response to each data point.</span></span> <span data-ttu-id="dee17-143">這是在機器人領域中的常見方法，其中在某個時間點的感應器讀數集就是一個資料點，而演算法必須選擇機器人的下一個動作。</span><span class="sxs-lookup"><span data-stu-id="dee17-143">It is a common approach in robotics, where the set of sensor readings at one point in time is a data point, and the algorithm must choose the robot’s next action.</span></span> <span data-ttu-id="dee17-144">它也很適合用於物聯網 (Internet of Things) 應用程式。</span><span class="sxs-lookup"><span data-stu-id="dee17-144">It's also a natural fit for Internet of Things applications.</span></span> <span data-ttu-id="dee17-145">此學習演算法也會在短時間內收到獎勵訊號，指出決策的好壞程度。</span><span class="sxs-lookup"><span data-stu-id="dee17-145">The learning algorithm also receives a reward signal a short time later, indicating how good the decision was.</span></span> <span data-ttu-id="dee17-146">演算法會據此修改其策略，以達到最高的獎勵。</span><span class="sxs-lookup"><span data-stu-id="dee17-146">Based on this, the algorithm modifies its strategy in order to achieve the highest reward.</span></span> <span data-ttu-id="dee17-147">Azure ML 中目前沒有增強式學習演算法模組。</span><span class="sxs-lookup"><span data-stu-id="dee17-147">Currently there are no reinforcement learning algorithm modules in Azure ML.</span></span>

* <span data-ttu-id="dee17-148">**貝氏法**可進行統計上獨立的資料點假設。</span><span class="sxs-lookup"><span data-stu-id="dee17-148">**Bayesian methods** make the assumption of statistically independent data points.</span></span> <span data-ttu-id="dee17-149">這表示在一個資料點中的非模型化變異不會與其他資料點相關，也就是，無法加以預測。</span><span class="sxs-lookup"><span data-stu-id="dee17-149">This means that the unmodeled variability in one data point is uncorrelated with others, that is, it can’t be predicted.</span></span> <span data-ttu-id="dee17-150">例如，如果要記錄的資料是下一班地鐵到達前的分鐘數，則相隔一天取得的兩個測量值屬於統計獨立。</span><span class="sxs-lookup"><span data-stu-id="dee17-150">For example, if the data being recorded is the number of minutes until the next subway train arrives, two measurements taken a day apart are statistically independent.</span></span> <span data-ttu-id="dee17-151">不過，相隔一分鐘取得的兩個測量值則不屬於統計獨立 - 其中一個值可高度預測另一個值。</span><span class="sxs-lookup"><span data-stu-id="dee17-151">However, two measurements taken a minute apart are not statistically independent - the value of one is highly predictive of the value of the other.</span></span>

* <span data-ttu-id="dee17-152">**促進式決策樹迴歸**會利用特徵重疊或特徵間的互動。</span><span class="sxs-lookup"><span data-stu-id="dee17-152">**Boosted decision tree regression** takes advantage of feature overlap or interaction among features.</span></span> <span data-ttu-id="dee17-153">這表示，在任何給定的資料點中，一項特徵的值有點能夠預測另一項特徵的值。</span><span class="sxs-lookup"><span data-stu-id="dee17-153">That means that, in any given data point, the value of one feature is somewhat predictive of the value of another.</span></span> <span data-ttu-id="dee17-154">例如，在每日高/低溫資料中，得知當天的低溫可讓您合理猜測高溫。</span><span class="sxs-lookup"><span data-stu-id="dee17-154">For example, in daily high/low temperature data, knowing the low temperature for the day allows you to make a reasonable guess at the high.</span></span> <span data-ttu-id="dee17-155">兩個特徵中包含的資訊有點冗餘。</span><span class="sxs-lookup"><span data-stu-id="dee17-155">The information contained in the two features is somewhat redundant.</span></span>

* <span data-ttu-id="dee17-156">使用原本就是多類別的分類器，或將一組雙類別的分類器合併成一個**整體** (Ensemble)，即可將資料分類成兩個以上的類別。</span><span class="sxs-lookup"><span data-stu-id="dee17-156">Classifying data into more than two categories can be done by either using an inherently multi-class classifier, or by combining a set of two-class classifiers into an **ensemble**.</span></span> <span data-ttu-id="dee17-157">在整體方法中，每個類別有個別的雙類別分類器 - 每個分類器會將資料分為兩類：「這一類」和「非這一類」。</span><span class="sxs-lookup"><span data-stu-id="dee17-157">In the ensemble approach, there is a separate two-class classifier for each class - each one separates the data into two categories:  “this class” and “not this class.”</span></span> <span data-ttu-id="dee17-158">然後這些分類器會表決資料點的正確指派。</span><span class="sxs-lookup"><span data-stu-id="dee17-158">Then these classifiers vote on the correct assignment of the data point.</span></span> <span data-ttu-id="dee17-159">這是[一對多的多類別][one-vs-all-multiclass]背後的運作原則。</span><span class="sxs-lookup"><span data-stu-id="dee17-159">This is the operational principle behind [One-vs-All Multiclass][one-vs-all-multiclass].</span></span>

* <span data-ttu-id="dee17-160">數種方法 (包括羅吉斯迴歸和貝氏點機器) 採用**線性類別界限**。</span><span class="sxs-lookup"><span data-stu-id="dee17-160">Several methods, including logistic regression and the Bayes point machine, assume **linear class boundaries**.</span></span> <span data-ttu-id="dee17-161">也就是說，它們會假設類別之間的界限大約是直線 (或者，更一般的情況下為超平面)。</span><span class="sxs-lookup"><span data-stu-id="dee17-161">That is, they assume that the boundaries between classes are approximately straight lines (or hyperplanes in the more general case).</span></span> <span data-ttu-id="dee17-162">通常這是您嘗試劃分之後才會知道的資料特性，但通常可以透過事先視覺化來學習。</span><span class="sxs-lookup"><span data-stu-id="dee17-162">Often this is a characteristic of the data that you don’t know until after you’ve tried to separate it, but it’s something that typically can be learned by visualizing beforehand.</span></span> <span data-ttu-id="dee17-163">如果類別界限看起來非常不規則，請堅守決策樹、決策叢林、支援向量機器或類神經網路。</span><span class="sxs-lookup"><span data-stu-id="dee17-163">If the class boundaries look very irregular, stick with decision trees, decision jungles, support vector machines, or neural networks.</span></span>

* <span data-ttu-id="dee17-164">為每個類別建立**虛擬變數**，並將它設定為 1 (適用此類別時) 或 0 (不適用此類別時)，即可搭配類別變數使用類神經網路。</span><span class="sxs-lookup"><span data-stu-id="dee17-164">Neural networks can be used with categorical variables by creating a **dummy variable** for each category, setting it to 1 in cases where the category applies, 0 where it doesn’t.</span></span>
