## <a name="multiple-network-paths"></a><span data-ttu-id="a9922-110">多個網路路徑</span><span class="sxs-lookup"><span data-stu-id="a9922-110">Multiple network paths</span></span>
<span data-ttu-id="a9922-111">當企業網路只有一個透過其網際網路服務提供者連至網際網路的連結時，則往返網際網路的所有流量都會經過相同的路徑。</span><span class="sxs-lookup"><span data-stu-id="a9922-111">When an enterprise network has only one link to the Internet through their Internet service provider, all traffic to and from the Internet travels the same path.</span></span> <span data-ttu-id="a9922-112">公司通常會購買多個線路，做為備援路徑，以改善網路運作時間。</span><span class="sxs-lookup"><span data-stu-id="a9922-112">Often, companies purchase multiple circuits, as redundant paths, to improve network uptime.</span></span> <span data-ttu-id="a9922-113">發生這種情況時，離開網路而朝向網際網路的流量可能會通過一個連結，而回傳流量則是通過不同的連結。</span><span class="sxs-lookup"><span data-stu-id="a9922-113">When this happens, it's possible that traffic that goes outside of the network, to the Internet, goes through one link, and the return traffic goes through a different link.</span></span> <span data-ttu-id="a9922-114">這通常稱為非對稱式路由。</span><span class="sxs-lookup"><span data-stu-id="a9922-114">This is commonly known as asymmetric routing.</span></span> <span data-ttu-id="a9922-115">在非對稱式路由中，反向網路流量會採取與原始流程不同的路徑。</span><span class="sxs-lookup"><span data-stu-id="a9922-115">In asymmetric routing, reverse network traffic takes a different path from the original flow.</span></span>
