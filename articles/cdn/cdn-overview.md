> [!NOTE]
> <span data-ttu-id="cd1a9-104">本文件說明 Azure 內容傳遞網路 (CDN) 是什麼、其運作方式與每項 Azure CDN 產品的功能。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-104">This document describes what the Azure Content Delivery Network (CDN) is, how it works, and the features of each Azure CDN product.</span></span>  <span data-ttu-id="cd1a9-105">如果您想要跳過這項資訊，直接進入如何建立 CDN 端點的教學課程，請參閱 [使用 Azure CDN](cdn-create-new-endpoint.md)。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-105">If you want to skip this information and go straight to a tutorial on how to create a CDN endpoint, see [Using Azure CDN](cdn-create-new-endpoint.md).</span></span>  <span data-ttu-id="cd1a9-106">如果您想要查看目前的 CDN 節點位置清單，請參閱 [Azure CDN POP 位置](cdn-pop-locations.md)。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-106">If you want to see a list of current CDN node locations, see [Azure CDN POP Locations](cdn-pop-locations.md).</span></span>

<span data-ttu-id="cd1a9-107">Azure 內容傳遞網路 (CDN) 會在策略性放置的位置上快取靜態 Web 內容，以提供最大輸送量來將內容傳遞給使用者。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-107">The Azure Content Delivery Network (CDN) caches static web content at strategically placed locations to provide maximum throughput for delivering content to users.</span></span>  <span data-ttu-id="cd1a9-108">CDN 為開發人員提供一套全球解決方案，以在全球實體節點上快取內容來傳遞高頻寬內容。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-108">The CDN offers developers a global solution for delivering high-bandwidth content by caching the content at physical nodes across the world.</span></span> 

<span data-ttu-id="cd1a9-109">使用 CDN 來快取網站資產的優點包括：</span><span class="sxs-lookup"><span data-stu-id="cd1a9-109">The benefits of using the CDN to cache web site assets include:</span></span>

* <span data-ttu-id="cd1a9-110">讓使用者享有更好的效能和使用者經驗，尤其是當使用的應用程式需要反覆存取多次才能載入內容時。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-110">Better performance and user experience for end users, especially when using applications where multiple round-trips are required to load content.</span></span>
* <span data-ttu-id="cd1a9-111">可進行大幅調整以更妥善地處理瞬間大量負載 (例如產品上市事件的開始)。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-111">Large scaling to better handle instantaneous high load, like at the start of a product launch event.</span></span>
* <span data-ttu-id="cd1a9-112">透過分散使用者要求以及從 Edge Server 提供內容，傳送至原始來源的流量將會減少。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-112">By distributing user requests and serving content from edge servers, less traffic is sent to the origin.</span></span>

## <a name="how-it-works"></a><span data-ttu-id="cd1a9-113">運作方式</span><span class="sxs-lookup"><span data-stu-id="cd1a9-113">How it works</span></span>
![CDN 概觀](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="cd1a9-115">使用者 (Alice) 使用具有特殊網域名稱的 URL (例如 `<endpointname>.azureedge.net`) 要求檔案 (也稱為資產)。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="cd1a9-116">DNS 將要求路由傳送到最佳的存在點 (POP) 位置。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="cd1a9-117">這通常是地理位置最接近使用者的 POP。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="cd1a9-118">如果 POP 中的 Edge Server 在其快取中沒有該檔案，Edge Server 便會從原始來源要求檔案。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="cd1a9-119">原始來源可以是 Azure Web 應用程式、Azure 雲端服務、Azure 儲存體帳戶或任何可公開存取的 Web 伺服器。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="cd1a9-120">原始來源將檔案傳回給 Edge Server，包括描述檔案存留時間 (TTL) 的選擇性 HTTP 標頭。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="cd1a9-121">Edge Server 快取檔案，並將檔案傳回給原始要求者 (Alice)。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="cd1a9-122">在 TTL 到期之前，檔案會一直快取在 Edge Server 上。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="cd1a9-123">如果原始來源未指定 TTL，預設 TTL 為 7 天。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="cd1a9-124">其他使用者後來可能會使用相同 URL 要求相同檔案，而且也可能導向至該相同 POP。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="cd1a9-125">如果檔案的 TTL 尚未過期，Edge Server 便會從快取傳回檔案。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="cd1a9-126">這會產生更快、更靈敏回應的使用者經驗。</span><span class="sxs-lookup"><span data-stu-id="cd1a9-126">This results in a faster, more responsive user experience.</span></span>
