### <a name="what-are-action-executions"></a><span data-ttu-id="85a13-111">什麼是動作執行？</span><span class="sxs-lookup"><span data-stu-id="85a13-111">What are action executions?</span></span>
<span data-ttu-id="85a13-112">在邏輯應用程式定義中的每個步驟都是動作，包括觸發程序、控制流程步驟 (如條件、範圍、for each 迴圈、do until 迴圈、呼叫連接器和呼叫原生動作)。</span><span class="sxs-lookup"><span data-stu-id="85a13-112">Every step in a logic app definition is an action, which includes triggers, control flow steps like conditions, scopes, for each loops, do until loops, calls to connectors and calls to native actions.</span></span>
<span data-ttu-id="85a13-113">觸發程序是設計用來在特定事件發生時，具現化新的邏輯應用程式執行個體的特殊動作。</span><span class="sxs-lookup"><span data-stu-id="85a13-113">Triggers are special actions that are designed to instantiate a new instance of a logic app when a particular event occurs.</span></span>  <span data-ttu-id="85a13-114">觸發程序擁有許多會影響邏輯應用程式計量方式的不同行為。</span><span class="sxs-lookup"><span data-stu-id="85a13-114">There are several different behaviors for triggers, which could affect how the logic app is metered.</span></span>
* <span data-ttu-id="85a13-115">**輪詢觸發程序** – 此觸發程序會持續輪詢端點，直到它收到的訊息符合用來建立邏輯應用程式執行個體的準則。</span><span class="sxs-lookup"><span data-stu-id="85a13-115">**Polling trigger** – this trigger continually polls an endpoint until it receives a message that satisfies the criteria for creating an instance of a logic app.</span></span>  <span data-ttu-id="85a13-116">在 Logic App Designer 中，可於觸發程序內設定輪詢間隔。</span><span class="sxs-lookup"><span data-stu-id="85a13-116">The polling interval can be configured in the trigger in the Logic App Designer.</span></span>  <span data-ttu-id="85a13-117">即使是未建立邏輯應用程式執行個體的輪詢要求也會計算為一個動作執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-117">Each polling request, even if it doesn’t create an instance of a logic app, counts as an action execution.</span></span>
* <span data-ttu-id="85a13-118">**Webhook 觸發程序** – 此觸發程序會等候用戶端送來針對特定端點的要求。</span><span class="sxs-lookup"><span data-stu-id="85a13-118">**Webhook trigger** – this trigger waits for a client to send it a request on a particular endpoint.</span></span>  <span data-ttu-id="85a13-119">每個傳送至 Webhook 端點的要求都會計算為一個動作執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-119">Each request sent to the webhook endpoint counts as an action execution.</span></span> <span data-ttu-id="85a13-120">要求觸發程序和 HTTP Webhook 觸發程序皆為 Webhook 觸發程序。</span><span class="sxs-lookup"><span data-stu-id="85a13-120">The Request and the HTTP Webhook trigger are both webhook triggers.</span></span>
* <span data-ttu-id="85a13-121">**循環觸發程序** – 此觸發程序會根據觸發程序中所設定的循環間隔建立邏輯應用程式執行個體。</span><span class="sxs-lookup"><span data-stu-id="85a13-121">**Recurrence trigger** – this trigger creates an instance of the logic app based on the recurrence interval configured in the trigger.</span></span>  <span data-ttu-id="85a13-122">例如，您可以將循環觸發程序設定為每三天執行一次，或甚至是每分鐘執行一次。</span><span class="sxs-lookup"><span data-stu-id="85a13-122">For example, a recurrence trigger can be configured to run every three days or even every minute.</span></span>

<span data-ttu-id="85a13-123">在 [Logic Apps 資源] 刀鋒視窗的 [觸發程序記錄] 部分可以看到觸發程序執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-123">Trigger executions can be seen in the Logic Apps resource blade in the Trigger History part.</span></span>

<span data-ttu-id="85a13-124">所有已執行的動作不論成功或失敗，都為計量為動作執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-124">All actions that were executed, whether they were successful or failed are metered as an action execution.</span></span>  <span data-ttu-id="85a13-125">因為條件不符而略過的動作，或因為邏輯應用程式在完成之前就終止而未執行的動作，均不會計算為一個動作執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-125">Actions that were skipped due to a condition not being met or actions that didn’t execute because the logic app terminated before completion are not counted as action executions.</span></span>

<span data-ttu-id="85a13-126">在迴圈內執行的動作每反覆運算一次就會計算一次。</span><span class="sxs-lookup"><span data-stu-id="85a13-126">Actions executed within loops are counted per iteration of the loop.</span></span>  <span data-ttu-id="85a13-127">例如，for each 迴圈內會針對 10 個項目所組成的清單逐一執行一遍的單一動作，將會計算為清單項目數 (10) 乘以迴圈中的動作數目 (1) 再加上起始迴圈時的一次執行，因此得出此範例共有 (10 * 1) + 1 = 11 個動作執行。</span><span class="sxs-lookup"><span data-stu-id="85a13-127">For example, a single action in a for each loop iterating through a list of 10 items are counted as the number of items in the list (10) multiplied by the number of actions in the loop (1) plus one for the initiation of the loop, which, in this example, would be (10 * 1) + 1 = 11 action executions.</span></span>
<span data-ttu-id="85a13-128">已停用的 Logic Apps 不能具現化新的執行個體，因此在停用時不會收費。</span><span class="sxs-lookup"><span data-stu-id="85a13-128">Disabled Logic Apps cannot have new instances instantiated and therefore, while disabled, are not charged.</span></span>  <span data-ttu-id="85a13-129">請注意，在停用邏輯應用程式之後，執行個體可能要先花點時間停止，然後才能完全停用。</span><span class="sxs-lookup"><span data-stu-id="85a13-129">Be mindful that after disabling a logic app it may take a little time for the instances to quiesce before being completely disabled.</span></span>
