# <a name="risk-assessment-ra"></a>風險評估 (RA)

> [!NOTE]
> 這些控制措施是由 NIST 與美國商務部 (U.S. Department of Commerce) 於 NIST 特刊 800-53 第 4 版 (NIST Special Publication 800-53 Revision 4) 中所定義。 如需了解每項控制措施的測試程序和指導方針相關資訊，請參閱《NIST 800-53 第 4 版》。

## <a name="nist-800-53-control-ra-1"></a>NIST 800-53 控制措施 RA-1

#### <a name="risk-assessment-policy-and-procedures"></a>風險評估原則和程序

**RA-1** 組織會針對 [指派：組織定義的人員或角色]，開發、記載及散佈因應目的、範圍、角色、職責、管理承諾、組織實體間之協調合作及合規性的風險評估原則，以及可協助實作風險評估原則和相關風險評估控制措施的程序；並檢閱和更新目前的風險評估原則 (依 [指派：組織定義的頻率]) 與風險評估程序 (依 [指派：組織定義的頻率])。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶的企業層級風險評估原則和程序可能即已足以因應這項控制措施。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-2a"></a>NIST 800-53 控制措施 RA-2.a

#### <a name="security-categorization"></a>安全性分類

**RA-2.a** 組織會根據適用的聯邦法律、行政命令、指示、原則、法規、標準及指導方針，將資訊和資訊系統分類。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶須負責將客戶部署的資源及其內含的資訊進行分類。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-2b"></a>NIST 800-53 控制措施 RA-2.b

#### <a name="security-categorization"></a>安全性分類

**RA-2.b** 組織會將安全性分類結果 (包括所依據的基本原理) 記載在資訊系統的安全性計畫中。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶須負責記載 RA-02.a 中定義的安全性分類結果。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-2c"></a>NIST 800-53 控制措施 RA-2.c

#### <a name="security-categorization"></a>安全性分類

**RA-2.c** 組織會確保授權官方人員或授權官方人員所指定的代表檢閱並核准安全性分類決策。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶須負責確保安全性分類決策通過檢閱及核准。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-3a"></a>NIST 800-53 控制措施 RA-3.a

#### <a name="risk-assessment"></a>風險評估

**RA-3.a** 組織會進行風險評估，包括就資訊系統及其處理、儲存或傳輸之資訊而言，未經授權的存取、使用、公開、中斷、修改或破壞所造成之損害的可能性與強度。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶須負責執行風險評估。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-3b"></a>NIST 800-53 控制措施 RA-3.b

**責任：**`Customer Only`

|---|---|

**責任：**`Customer Only`

 ## <a name="nist-800-53-control-ra-3e"></a>NIST 800-53 控制措施 RA-3.e

#### <a name="risk-assessment"></a>風險評估

**RA-3.e** 組織會依 [指派：組織定義的頻率] 或每當資訊系統或作業環境有重大變更時 (包括識別出新的威脅或弱點)，或是發生其他可能影響系統安全性狀態的情況時，更新風險評估。

**責任：**`Customer Only`

 ## <a name="nist-800-53-control-ra-5a"></a>NIST 800-53 控制措施 RA-5.a

#### <a name="vulnerability-scanning"></a>弱點掃描

**RA-5.a** 組織會依 [指派：組織定義的頻率和/或隨機 (根據組織定義的程序)]，以及在識別出並回報有可能會影響系統/應用程式的新弱點時，掃描資訊系統和所裝載應用程式中的弱點。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 這個 Azure Blueprint (藍圖) 會部署「OMS 安全性與稽核」解決方案。 這個解決方案可提供安全性狀態的全面性檢視。 此解決方案提供兩個安全性網域，亦即「更新評估」和「基準評估」。 「基準評估」會評估一組登錄機碼、稽核原則設定和安全性原則設定，以及 Microsoft 針對這些設定的建議值。 「更新評估」會評估所有已部署虛擬機器上可用更新的狀態。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-5b"></a>NIST 800-53 控制措施 RA-5.b

#### <a name="vulnerability-scanning"></a>弱點掃描

**RA-5.b** 組織會採用弱點掃描工具和技術，透過使用標準 (用於列舉平台、軟體缺陷及不適當組態；將檢查清單與測試程序格式化；以及評估弱點影響) 來協助工具間的互通性，以及將弱點管理程序的各個部分自動化。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 這個 Azure Blueprint (藍圖) 會部署「OMS 安全性與稽核」解決方案。 此解決方案提供兩個安全性網域，亦即「更新評估」和「基準評估」。 系統會為已識別的問題指派一個嚴重性等級。 針對「基準評估」規則，會提供「通用組態列舉」(Common Configuration Enumeration, CCE) 識別碼。  |
| **提供者 (Microsoft Azure)** | 不適用 |

 ## <a name="nist-800-53-control-ra-5d"></a>NIST 800-53 控制措施 RA-5.d

#### <a name="vulnerability-scanning"></a>弱點掃描

**RA-5.d** 組織會根據組織的風險評估，依 [指派：組織定義的回應時間] 補救合法弱點。

| **提供者 (Microsoft Azure)** | 不適用 |


 ## <a name="nist-800-53-control-ra-5e"></a>NIST 800-53 控制措施 RA-5.e

#### <a name="vulnerability-scanning"></a>弱點掃描

**RA-5.e** 組織會將從弱點掃描程序及安全性控制評估取得的資訊與 [指派：組織定義的人員或角色] 共用，以協助排除其他資訊系統中類似的弱點 (亦即系統弱點或不足之處)。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 客戶須負責共用從弱點掃描程序及安全性控制評估取得的資訊，以協助排除所有客戶部署資源中類似的弱點。 |
| **提供者 (Microsoft Azure)** | 不適用 |


 ### <a name="nist-800-53-control-ra-5-1"></a>NIST 800-53 控制措施 RA-5 (1)

**RA-5 (1)** 組織會採用能夠隨時更新所要掃描之資訊系統弱點的弱點掃描工具。

| **提供者 (Microsoft Azure)** | 不適用 |

#### <a name="vulnerability-scanning--update-by-frequency--prior-to-new-scan--when-identified"></a>弱點掃描 | 依頻率/在進行新的掃描之前/在識別時進行更新

**RA-5 (2)** 組織會 [選取項目 (一或多個)：依 [指派：組織定義的頻率]；在進行新的掃描之前；在識別出並回報新的弱點時]，更新所掃描到的資訊系統弱點。

| **提供者 (Microsoft Azure)** | 不適用 |


 ### <a name="nist-800-53-control-ra-5-3"></a>NIST 800-53 控制措施 RA-5 (3)

#### <a name="vulnerability-scanning--breadth--depth-of-coverage"></a>弱點掃描 | 涵蓋範圍的寬度/深度

**RA-5 (3)** 組織會採用可識別涵蓋範圍寬度和深度 (亦即所掃描的資訊系統元件和所檢查的弱點) 的弱點掃描程序。

**責任：**`Customer Only`

|||
|---|---|
| **客戶** | 這個 Azure Blueprint (藍圖) 會部署「OMS 安全性與稽核」解決方案。 此解決方案提供兩個安全性網域，亦即「更新評估」和「基準評估」。 更新評估會同時對 Windows 和 Linux 虛擬機器執行評估。 基準評估會根據業界最佳做法，使用 Microsoft 所開發的基準，對 Windows 虛擬機器執行評估。 |
| **提供者 (Microsoft Azure)** | 不適用 |
