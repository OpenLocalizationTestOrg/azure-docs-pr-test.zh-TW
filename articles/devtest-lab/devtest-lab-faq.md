## <a name="what-are-the-different-security-levels-in-azure-devtest-labs"></a><span data-ttu-id="be291-136">Azure DevTest Labs 中有哪些不同的安全性層級？</span><span class="sxs-lookup"><span data-stu-id="be291-136">What are the different security levels in Azure DevTest Labs?</span></span>
<span data-ttu-id="be291-137">安全性存取權是由 [Azure 角色型存取控制 (RBAC)](../active-directory/role-based-access-built-in-roles.md)所決定。</span><span class="sxs-lookup"><span data-stu-id="be291-137">Security access is determined by [Azure Role-Based Access Control (RBAC)](../active-directory/role-based-access-built-in-roles.md).</span></span> <span data-ttu-id="be291-138">若要了解存取權的運作方式，了解 RBAC 所定義的權限、角色和範圍之間的差異將有所幫助。</span><span class="sxs-lookup"><span data-stu-id="be291-138">To understand how access works, it helps to understand the differences between a permission, a role, and a scope as defined by RBAC.</span></span>

* <span data-ttu-id="be291-139">**權限** - 權限是針對特定動作所定義的存取權。</span><span class="sxs-lookup"><span data-stu-id="be291-139">**Permission** - A permission is a defined access to a specific action.</span></span> <span data-ttu-id="be291-140">例如，權限可以是對所有虛擬機器的讀取權限。</span><span class="sxs-lookup"><span data-stu-id="be291-140">For example, a permission can be read-access to all virtual machines.</span></span>
* <span data-ttu-id="be291-141">**角色** - 角色是一組可以分組並指派給使用者的權限。</span><span class="sxs-lookup"><span data-stu-id="be291-141">**Role** - A role is a set of permissions that can be grouped and assigned to a user.</span></span> <span data-ttu-id="be291-142">例如，「訂用帳戶擁有者」擁有訂用帳戶內所有資源的存取權。</span><span class="sxs-lookup"><span data-stu-id="be291-142">For example, a "subscription owner" has access to all resources within a subscription.</span></span>
* <span data-ttu-id="be291-143">**範圍** - 範圍是 Azure 資源階層中的層級。</span><span class="sxs-lookup"><span data-stu-id="be291-143">**Scope** - A scope is a level within the hierarchy of Azure resource.</span></span> <span data-ttu-id="be291-144">例如，範圍可以是一個資源群組、單一實驗室或整個訂用帳戶。</span><span class="sxs-lookup"><span data-stu-id="be291-144">For example, a scope can be a resource group or a single lab or the entire subscription.</span></span>

<span data-ttu-id="be291-145">在 Azure DevTest Labs 的範圍內有兩種可用來定義使用者權限的角色類型︰實驗室擁有者和實驗室使用者。</span><span class="sxs-lookup"><span data-stu-id="be291-145">Within the scope of Azure DevTest Labs, there are two types of roles to define user permissions: lab owner and lab user.</span></span>

* <span data-ttu-id="be291-146">**實驗室擁有者** - 實驗室擁有者擁有實驗室內任何資源的存取權。</span><span class="sxs-lookup"><span data-stu-id="be291-146">**Lab owner** - A lab owner has the access to any resources within the lab.</span></span> <span data-ttu-id="be291-147">因此，他們可以修改原則、讀取和寫入任何 VM、變更虛擬網路等等。</span><span class="sxs-lookup"><span data-stu-id="be291-147">Therefore, they can modify policies, read and write any VMs, change the virtual network, and so on.</span></span>
* <span data-ttu-id="be291-148">**實驗室使用者** - 實驗室使用者可以檢視所有實驗室資源，例如 VM、原則和虛擬網路，但他們不能修改原則或任何由其他使用者所建立的 VM。</span><span class="sxs-lookup"><span data-stu-id="be291-148">**Lab user** - A lab user can view all lab resources, such as VMs, policies, and virtual networks, but they cannot modify policies or any VMs created by other users.</span></span> <span data-ttu-id="be291-149">您也可以在 Azure DevTest Labs 建立自訂角色，若要了解如何進行，請參閱 [將特定實驗室原則的權限授與使用者](devtest-lab-grant-user-permissions-to-specific-lab-policies.md)一文。</span><span class="sxs-lookup"><span data-stu-id="be291-149">It is also possible to create custom roles in Azure DevTest Labs, and you can learn how to do in the article, [Grant user permissions to specific lab policies](devtest-lab-grant-user-permissions-to-specific-lab-policies.md).</span></span>

<span data-ttu-id="be291-150">由於範圍是階層形式，當使用者擁有特定範圍的權限時，就會自動獲得該範圍所包含的每個較低層級範圍的權限。</span><span class="sxs-lookup"><span data-stu-id="be291-150">Since scopes are hierarchical, when a user has permissions at a certain scope, they are automatically granted those permissions at every lower-level scope encompassed.</span></span> <span data-ttu-id="be291-151">例如，如果有使用者指派給訂用帳戶擁有者角色，他們就可以存取訂用帳戶中的所有資源。</span><span class="sxs-lookup"><span data-stu-id="be291-151">For instance, if a user is assigned to the role of subscription owner, then they have access to all resources in a subscription.</span></span> <span data-ttu-id="be291-152">這些資源包括所有虛擬機器、所有虛擬網路和所有實驗室。</span><span class="sxs-lookup"><span data-stu-id="be291-152">These resources include all virtual machines, all virtual networks, and all labs.</span></span> <span data-ttu-id="be291-153">因此，訂用帳戶擁有者會自動繼承實驗室擁有者角色。</span><span class="sxs-lookup"><span data-stu-id="be291-153">Thus, a subscription owner automatically inherits the role of lab owner.</span></span> <span data-ttu-id="be291-154">不過，若情形顛倒過來就不成立。</span><span class="sxs-lookup"><span data-stu-id="be291-154">However, the opposite is not true.</span></span> <span data-ttu-id="be291-155">實驗室擁有者可存取實驗室，而實驗室是比訂用帳戶層級還低的範圍。</span><span class="sxs-lookup"><span data-stu-id="be291-155">A lab owner has access to a lab, which is a lower scope than the subscription level.</span></span> <span data-ttu-id="be291-156">因此，實驗室擁有者將無法看到實驗室以外的虛擬機器、虛擬網路或任何資源。</span><span class="sxs-lookup"><span data-stu-id="be291-156">Therefore, a lab owner is not able to see virtual machines or virtual networks or any resources that are outside of the lab.</span></span>

## <a name="how-do-i-create-a-role-to-allow-users-to-perform-a-specific-task"></a><span data-ttu-id="be291-157">如何建立可讓使用者執行特定工作的角色？</span><span class="sxs-lookup"><span data-stu-id="be291-157">How do I create a role to allow users to perform a specific task?</span></span>
<span data-ttu-id="be291-158">您可以在這裡找到有關如何建立自訂角色並指派權限給該角色的完整文章。</span><span class="sxs-lookup"><span data-stu-id="be291-158">A comprehensive article about how to create custom roles and assign permissions to that role can be found here.</span></span> <span data-ttu-id="be291-159">以下指令碼範例會建立「DevTest Labs 進階使用者」角色，其具有啟動和停止實驗室中所有 VM 的權限︰</span><span class="sxs-lookup"><span data-stu-id="be291-159">Here is an example of a script that creates the role "DevTest Labs Advanced User", which has permission to start and stop all VMs in the lab:</span></span>

    $policyRoleDef = Get-AzureRmRoleDefinition "DevTest Labs User"
    $policyRoleDef.Actions.Remove('Microsoft.DevTestLab/Environments/*')

    $policyRoleDef.IsCustom = $true
    $policyRoleDef.AssignableScopes.Clear()
    $policyRoleDef.AssignableScopes.Add("subscriptions/<subscription Id>")
    $policyRoleDef.Actions.Add("Microsoft.DevTestLab/labs/virtualMachines/Start/action")
    $policyRoleDef.Actions.Add("Microsoft.DevTestLab/labs/virtualMachines/Stop/action")
    $policyRoleDef = New-AzureRmRoleDefinition -Role $policyRoleDef
