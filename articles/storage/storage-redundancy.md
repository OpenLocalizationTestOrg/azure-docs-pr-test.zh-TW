## <a name="geo-redundant-storage"></a><span data-ttu-id="d2a1a-170">異地備援儲存體</span><span class="sxs-lookup"><span data-stu-id="d2a1a-170">Geo-redundant storage</span></span>

<span data-ttu-id="d2a1a-173">在已啟用 GRS 的儲存體帳戶中，更新會先交付到主要區域，並在此複寫三次。</span><span class="sxs-lookup"><span data-stu-id="d2a1a-173">For a storage account with GRS enabled, an update is first committed to the primary region, where it is replicated three times.</span></span> <span data-ttu-id="d2a1a-174">然後更新會以非同步的方式複寫到次要地區，並同樣在此複寫三次。</span><span class="sxs-lookup"><span data-stu-id="d2a1a-174">Then the update is replicated asynchronously to the secondary region, where it is also replicated three times.</span></span>

<span data-ttu-id="d2a1a-175">使用 GRS 時，主要和次要區域會管理分散在儲存體縮放單位內不同容錯網域和升級網域之間的複本，如同 LRS 的描述。</span><span class="sxs-lookup"><span data-stu-id="d2a1a-175">With GRS, both the primary and secondary regions manage replicas across separate fault domains and upgrade domains within a storage scale unit as described with LRS.</span></span>
