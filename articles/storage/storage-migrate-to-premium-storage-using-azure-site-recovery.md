<span data-ttu-id="53ef5-113">我們的建議是使用 Site Recovery 來移轉到進階儲存體，因為此選項的停機時間最短，並且不用手動執行複製磁碟和建立新 VM 的作業。</span><span class="sxs-lookup"><span data-stu-id="53ef5-113">We recommend migrating to Premium Storage by using Site Recovery because this option provides minimal downtime and avoids the manual execution of copying disks and creating new VMs.</span></span> <span data-ttu-id="53ef5-114">在容錯移轉期間，Site Recovery 會有系統地複製您的磁碟並建立新的 VM。</span><span class="sxs-lookup"><span data-stu-id="53ef5-114">Site Recovery will systematically copy your disks and create new VMs during failover.</span></span> <span data-ttu-id="53ef5-115">Site Recovery 支援數種類型的容錯移轉，且停機時間最短或甚至不用停機。</span><span class="sxs-lookup"><span data-stu-id="53ef5-115">Site Recovery supports a number of types of failover with minimal or no downtime.</span></span> <span data-ttu-id="53ef5-116">若要規劃停機時間並預估資料損失情形，請參閱 Site Recovery 中的[容錯移轉的類型](../site-recovery/site-recovery-failover.md)資料表。</span><span class="sxs-lookup"><span data-stu-id="53ef5-116">To plan your downtime and estimate data loss, see the [Types of failover](../site-recovery/site-recovery-failover.md) table in Site Recovery.</span></span> <span data-ttu-id="53ef5-117">如果您[準備在容錯移轉後連接到 Azure VM](../site-recovery/site-recovery-vmware-to-azure.md)，您應該可以在容錯移轉後使用 RDP 連接到 Azure VM。</span><span class="sxs-lookup"><span data-stu-id="53ef5-117">If you [prepare to connect to Azure VMs after failover](../site-recovery/site-recovery-vmware-to-azure.md), you should be able to connect to the Azure VM using RDP after failover.</span></span>
