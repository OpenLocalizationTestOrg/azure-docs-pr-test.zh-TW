<span data-ttu-id="86c97-170">標記計畫必須像是使用者路徑或一種應用程式工作流程的描述，提供所有必要標記 (資料)，必須收集這些標記才能進行足夠的分析，了解使用者行為，並正確區隔使用者基礎。</span><span class="sxs-lookup"><span data-stu-id="86c97-170">A tag plan must be like a description of the user-path or a kind of workflow of the application, providing all the necessary tags (data) that must be collected to have enough analytics to understand user behavior and properly segment the user base.</span></span> <span data-ttu-id="86c97-171">這不是技術性的程序。</span><span class="sxs-lookup"><span data-stu-id="86c97-171">This is not a technical process.</span></span> <span data-ttu-id="86c97-172">因此，行鎖人員能夠根據他們的 Mobile Engagement 策略指定想要收集的資料。</span><span class="sxs-lookup"><span data-stu-id="86c97-172">Hence, marketers are able to specify the data they want to collect based on their Mobile Engagement strategy.</span></span>

<span data-ttu-id="86c97-173">最低限度是至少標記應用程式的所有畫面 (在 Mobile Engagement 中稱為「活動」  )。</span><span class="sxs-lookup"><span data-stu-id="86c97-173">The minimum is to tag at least all the screens (called *activities* in Mobile Engagement) of an application.</span></span> <span data-ttu-id="86c97-174">如此將有助於判斷使用者路徑。</span><span class="sxs-lookup"><span data-stu-id="86c97-174">This helps determine the user-path.</span></span>

<span data-ttu-id="86c97-175">活動可以內嵌收集動作資訊的「事件」  ，例如按一下按鈕。</span><span class="sxs-lookup"><span data-stu-id="86c97-175">An activity can embed *events* that collect action information like clicking a button.</span></span> <span data-ttu-id="86c97-176">這麼做可收集應用程式內的互動資訊。</span><span class="sxs-lookup"><span data-stu-id="86c97-176">This allows the collection of interactions within the application.</span></span> <span data-ttu-id="86c97-177">於是，行銷人員便能得知使用者正在瀏覽的畫面，以及他們正在做的事。</span><span class="sxs-lookup"><span data-stu-id="86c97-177">Therefore, marketers are able to know what screen users are visiting and what they are doing.</span></span>

<span data-ttu-id="86c97-178">`Jobs` 是一段持續時間的動作。</span><span class="sxs-lookup"><span data-stu-id="86c97-178">`Jobs` are actions with a duration.</span></span> <span data-ttu-id="86c97-179">例如，這很適合讓行銷人員了解使用者在建立帳戶或登入時，用了多少時間。</span><span class="sxs-lookup"><span data-stu-id="86c97-179">This is very useful for marketer to understand how long it takes for a user to create an account or to login for instance.</span></span> <span data-ttu-id="86c97-180">這可能也很適合讓開發人員監視呼叫 Web 服務要花多少時間。</span><span class="sxs-lookup"><span data-stu-id="86c97-180">This also could be useful for developers to monitor how long it takes to call a web service.</span></span>

<span data-ttu-id="86c97-181">`Errors` 。</span><span class="sxs-lookup"><span data-stu-id="86c97-181">`Errors` can also be monitored to know if users are having issues in your app.</span></span> <span data-ttu-id="86c97-182">例如經常遇到連線問題。</span><span class="sxs-lookup"><span data-stu-id="86c97-182">For example, getting frequent connection issues.</span></span>

<span data-ttu-id="86c97-183">所有這類型的資料可以利用參數來增加 (Mobile Engagement 中的「extra-information」 )，讓您可以從應用程式收集動態資料。</span><span class="sxs-lookup"><span data-stu-id="86c97-183">All of this type of data can be augmented with parameters (*extra-information* in Mobile Engagement) allowing you to gather dynamic data from the application.</span></span> <span data-ttu-id="86c97-184">這很重要，因為可以進行精細的客群區隔。</span><span class="sxs-lookup"><span data-stu-id="86c97-184">This is important to allow fine-grained segmentation.</span></span> <span data-ttu-id="86c97-185">例如，行銷人員可以根據使用者取用的內容類型來區隔使用者。</span><span class="sxs-lookup"><span data-stu-id="86c97-185">For instance, marketers can segment a user based on the type of content they have consumed.</span></span> <span data-ttu-id="86c97-186">內容類型會是活動或事件的動態資訊。</span><span class="sxs-lookup"><span data-stu-id="86c97-186">The type of content will be the dynamic information of an activity or an event.</span></span>

<span data-ttu-id="86c97-187">*App-information* 是可讓您即時確認應用程式或使用者狀態的資料。</span><span class="sxs-lookup"><span data-stu-id="86c97-187">*App-information* is data that allows you to confirm the status of the application or of the user in real-time.</span></span> <span data-ttu-id="86c97-188">這也有助於分類對象族群，並快速鎖定目標。</span><span class="sxs-lookup"><span data-stu-id="86c97-188">This also helps to categorize an audience base and target it quickly.</span></span> <span data-ttu-id="86c97-189">例如，此項目可以利用 true/false 狀態，讓您知道使用者是否登入，或其訂用帳戶是否到期。</span><span class="sxs-lookup"><span data-stu-id="86c97-189">For example, it can use a true/false status of whether the user is logging in or not, or his subscription expiration date.</span></span>
