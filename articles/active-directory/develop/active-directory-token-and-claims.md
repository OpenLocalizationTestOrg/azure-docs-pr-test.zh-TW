# <a name="azure-ad-token-reference"></a><span data-ttu-id="61913-103">Azure AD 權杖參考</span><span class="sxs-lookup"><span data-stu-id="61913-103">Azure AD token reference</span></span>
<span data-ttu-id="61913-104">Azure Active Directory (Azure AD) 會在處理每個驗證流程時發出數種安全性權杖。</span><span class="sxs-lookup"><span data-stu-id="61913-104">Azure Active Directory (Azure AD) emits several types of security tokens in the processing of each authentication flow.</span></span> <span data-ttu-id="61913-105">本文件說明每種權杖的格式、安全性特性和內容。</span><span class="sxs-lookup"><span data-stu-id="61913-105">This document describes the format, security characteristics, and contents of each type of token.</span></span>

## <a name="types-of-tokens"></a><span data-ttu-id="61913-106">權杖的類型</span><span class="sxs-lookup"><span data-stu-id="61913-106">Types of tokens</span></span>
<span data-ttu-id="61913-107">Azure AD 支援 [OAuth 2.0 授權通訊協定](active-directory-protocols-oauth-code.md)，該通訊協定會使用 access_tokens 和 refresh_tokens。</span><span class="sxs-lookup"><span data-stu-id="61913-107">Azure AD supports the [OAuth 2.0 authorization protocol](active-directory-protocols-oauth-code.md), which makes use of both access_tokens and refresh_tokens.</span></span>  <span data-ttu-id="61913-108">它也支援透過 [OpenID Connect](active-directory-protocols-openid-connect-code.md) 進行驗證和登入，這引進第三種類型的權杖 - id_token。</span><span class="sxs-lookup"><span data-stu-id="61913-108">It also supports authentication and sign-in via [OpenID Connect](active-directory-protocols-openid-connect-code.md), which introduces a third type of token, the id_token.</span></span>  <span data-ttu-id="61913-109">每個權杖都是以「持有人權杖」表示。</span><span class="sxs-lookup"><span data-stu-id="61913-109">Each of these tokens is represented as a "bearer token".</span></span>

<span data-ttu-id="61913-110">持有人權杖是輕巧型的安全性權杖，授權「持有者」存取受保護的資源。</span><span class="sxs-lookup"><span data-stu-id="61913-110">A bearer token is a lightweight security token that grants the “bearer” access to a protected resource.</span></span> <span data-ttu-id="61913-111">從這個意義上說，「持有者」是可出示權杖的任何一方。</span><span class="sxs-lookup"><span data-stu-id="61913-111">In this sense, the “bearer” is any party that can present the token.</span></span> <span data-ttu-id="61913-112">雖然必須向 Azure AD 驗證才能接收持有人權杖，但必須採取步驟來保護權杖，以避免遭非預期人士攔截。</span><span class="sxs-lookup"><span data-stu-id="61913-112">Though authentication with Azure AD is required in order to receive a bearer token, steps must be taken to secure the token, to prevent interception by an unintended party.</span></span> <span data-ttu-id="61913-113">因為持有人權杖沒有內建的機制可防止未經授權的人士使用權杖，必須在安全通道，例如傳輸層安全性 (HTTPS) 來傳輸它們。</span><span class="sxs-lookup"><span data-stu-id="61913-113">Because bearer tokens do not have a built-in mechanism to prevent unauthorized parties from using them, they must be transported in a secure channel such as transport layer security (HTTPS).</span></span> <span data-ttu-id="61913-114">如果以純文字傳輸持有人權杖，便可能利用攔截式攻擊來取得權杖，然後取得對受保護的資源未經授權的存取。</span><span class="sxs-lookup"><span data-stu-id="61913-114">If a bearer token is transmitted in the clear, a man-in the middle attack can be used to acquire the token and gain unauthorized access to a protected resource.</span></span> <span data-ttu-id="61913-115">儲存或快取持有者權杖供以後使用時，也適用相同的安全性原則。</span><span class="sxs-lookup"><span data-stu-id="61913-115">The same security principles apply when storing or caching bearer tokens for later use.</span></span> <span data-ttu-id="61913-116">務必確定您的應用程式以安全的方式傳輸和儲存持有人權杖。</span><span class="sxs-lookup"><span data-stu-id="61913-116">Always ensure that your app transmits and stores bearer tokens in a secure manner.</span></span> <span data-ttu-id="61913-117">關於持有者權杖的其他安全性考量，請參閱 [RFC 6750 第 5 節](http://tools.ietf.org/html/rfc6750)。</span><span class="sxs-lookup"><span data-stu-id="61913-117">For more security considerations on bearer tokens, see [RFC 6750 Section 5](http://tools.ietf.org/html/rfc6750).</span></span>

<span data-ttu-id="61913-118">許多由 Azure AD 所簽發的權杖都會實作為 JSON Web 權杖或 JWT。</span><span class="sxs-lookup"><span data-stu-id="61913-118">Many of the tokens issued by Azure AD are implemented as JSON Web Tokens, or JWTs.</span></span>  <span data-ttu-id="61913-119">JWT 是一種精簡的 URL 安全方法，可在兩方之間傳輸資訊。</span><span class="sxs-lookup"><span data-stu-id="61913-119">A JWT is a compact, URL-safe means of transferring information between two parties.</span></span>  <span data-ttu-id="61913-120">JWT 中包含的資訊也稱為權杖持有人及主體相關資訊的「宣告」或判斷提示。</span><span class="sxs-lookup"><span data-stu-id="61913-120">The information contained in JWTs are known as "claims", or assertions of information about the bearer and subject of the token.</span></span>  <span data-ttu-id="61913-121">JWT 中的宣告是為了傳輸而編碼和序列化的 JSON 物件。</span><span class="sxs-lookup"><span data-stu-id="61913-121">The claims in JWTs are JSON objects encoded and serialized for transmission.</span></span>  <span data-ttu-id="61913-122">因為 Azure AD 所簽發的 JWT 已簽署但未加密，所以您可以輕鬆地檢查 JWT 的內容以便偵錯。</span><span class="sxs-lookup"><span data-stu-id="61913-122">Since the JWTs issued by Azure AD are signed, but not encrypted, you can easily inspect the contents of a JWT for debugging purposes.</span></span>  <span data-ttu-id="61913-123">有數個工具可以進行這項操作，例如 [jwt.calebb.net](http://jwt.calebb.net)。</span><span class="sxs-lookup"><span data-stu-id="61913-123">There are several tools available for doing so, such as [jwt.calebb.net](http://jwt.calebb.net).</span></span> <span data-ttu-id="61913-124">如需 JWT 的詳細資訊，您可以參考 [JWT 規格](http://self-issued.info/docs/draft-ietf-oauth-json-web-token.html)。</span><span class="sxs-lookup"><span data-stu-id="61913-124">For more information on JWTs, you can refer to the [JWT specification](http://self-issued.info/docs/draft-ietf-oauth-json-web-token.html).</span></span>

## <a name="idtokens"></a><span data-ttu-id="61913-125">Id_tokens</span><span class="sxs-lookup"><span data-stu-id="61913-125">Id_tokens</span></span>
<span data-ttu-id="61913-126">Id_tokens 是您的應用程式使用 [OpenID Connect](active-directory-protocols-openid-connect-code.md) 執行驗證時收到的一種登入安全性權杖形式。</span><span class="sxs-lookup"><span data-stu-id="61913-126">Id_tokens are a form of sign-in security token that your app receives when performing authentication using [OpenID Connect](active-directory-protocols-openid-connect-code.md).</span></span>  <span data-ttu-id="61913-127">其以 [JWT](#types-of-tokens)表示，而且包含可用於讓使用者登入您的應用程式的宣告。</span><span class="sxs-lookup"><span data-stu-id="61913-127">They are represented as [JWTs](#types-of-tokens), and contain claims that you can use for signing the user into your app.</span></span>  <span data-ttu-id="61913-128">您可以適時使用 id_token 中的宣告 - 通常用來顯示顯示帳戶資訊或在應用程式中進行存取控制決策。</span><span class="sxs-lookup"><span data-stu-id="61913-128">You can use the claims in an id_token as you see fit - commonly they are used for displaying account information or making access control decisions in an app.</span></span>

<span data-ttu-id="61913-129">Id_token 已簽署，但這次不加密。</span><span class="sxs-lookup"><span data-stu-id="61913-129">Id_tokens are signed, but not encrypted at this time.</span></span>  <span data-ttu-id="61913-130">當您的應用程式收到 id_token 時，它必須[驗證簽章](#validating-tokens)以證明權杖的真實性，以及驗證權杖中的幾個宣告來證明其有效性。</span><span class="sxs-lookup"><span data-stu-id="61913-130">When your app receives an id_token, it must [validate the signature](#validating-tokens) to prove the token's authenticity and validate a few claims in the token to prove its validity.</span></span>  <span data-ttu-id="61913-131">應用程式所驗證的宣告會視案例需求而有所不同，但您的應用程式必須在每一種案例中執行一些 [常見的宣告驗證](#validating-tokens) 。</span><span class="sxs-lookup"><span data-stu-id="61913-131">The claims validated by an app vary depending on scenario requirements, but there are some [common claim validations](#validating-tokens) that your app must perform in every scenario.</span></span>

<span data-ttu-id="61913-132">如需 Id_tokens 宣告以及 id_token 的相關範例，請參閱下一節。</span><span class="sxs-lookup"><span data-stu-id="61913-132">See the following section for information on id_tokens claims, as well as a sample id_token.</span></span>  <span data-ttu-id="61913-133">請注意，id_token 中的宣告不依任何特定順序傳回。</span><span class="sxs-lookup"><span data-stu-id="61913-133">Note that the claims in id_tokens are not returned in any particular order.</span></span>  <span data-ttu-id="61913-134">此外，新的宣告可以隨時引進 id_token 中 - 您的應用程式不會在引進新的宣告時中斷。</span><span class="sxs-lookup"><span data-stu-id="61913-134">In addition, new claims can be introduced into id_tokens at any point in time - your app should not break as new claims are introduced.</span></span>  <span data-ttu-id="61913-135">下列清單包含本文撰寫時您的應用程式可確實解譯的宣告。</span><span class="sxs-lookup"><span data-stu-id="61913-135">The following list includes the claims that your app can reliably interpret at the time of this writing.</span></span>  <span data-ttu-id="61913-136">如有需要，在 [OpenID Connect 規格](http://openid.net/specs/openid-connect-core-1_0.html)中可找到更多詳細資料。</span><span class="sxs-lookup"><span data-stu-id="61913-136">If necessary, even more detail can be found in the [OpenID Connect specification](http://openid.net/specs/openid-connect-core-1_0.html).</span></span>
